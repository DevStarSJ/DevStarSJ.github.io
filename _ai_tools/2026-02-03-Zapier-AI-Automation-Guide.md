## What is Zapier?

Zapier is a no-code automation platform that connects your apps and automates workflows. At its core, it uses "Zaps"—automated workflows that connect triggers (events that start workflows) to actions (tasks that get executed).

### What's New in 2026

- **AI by Zapier**: Built-in AI actions for any Zap
- **Chatbots**: Create custom AI assistants
- **Interfaces**: Build full web apps without code
- **Tables**: Native database with AI features
- **Canvas**: Visual workflow builder with AI
- **Central**: Team automation hub

## Core Concepts

### Triggers & Actions

**Trigger**: The event that starts your Zap
- New email in Gmail
- Form submission
- New Slack message
- Scheduled time

**Action**: What happens when triggered
- Create task in Asana
- Send email
- Update spreadsheet
- Post to Slack

### Multi-Step Zaps

Chain multiple actions together:
```
Trigger: New customer in Stripe
↓
Action 1: Create contact in HubSpot
↓
Action 2: Add to Mailchimp list
↓
Action 3: Send welcome Slack message
↓
Action 4: Create Notion page
```

## Advanced Features

### Paths (Conditional Logic)

Create branching workflows:

```
Trigger: New form response
↓
Path A: If "Enterprise" plan selected
  → Assign to enterprise sales team
  → Send enterprise welcome pack
↓
Path B: If "Starter" plan selected
  → Add to self-serve onboarding
  → Send tutorial emails
```

### Filters

Only continue if conditions are met:

```
Trigger: New Slack message
↓
Filter: Only continue if message contains "urgent"
↓
Action: Send push notification
```

### Formatter

Transform data between steps:

- **Text**: Extract, replace, truncate
- **Numbers**: Math operations, formatting
- **Dates**: Parse, format, calculate differences
- **Lookup Tables**: Map values

### Loops (Looping by Zapier)

Process multiple items:

```
Trigger: New spreadsheet row
↓
Action: Get all items from row
↓
Loop: For each item
  → Create individual task
```

### Schedule

Time-based automations:

- Run daily, weekly, monthly
- Specific times
- Cron expressions

## Pro Tips

### 1. Use Webhooks for Custom Integrations

When an app isn't supported:
```
Trigger: Webhook (catch)
→ Send from any app with HTTP capability
```

### 2. Error Handling

Set up error alerts:
```
When Zap fails → Send Slack notification with details
```

### 3. Use Delay Steps

Don't overwhelm systems:
```
Action 1 → Delay 5 minutes → Action 2
```

### 4. Version Control

Use naming conventions:
```
[Department] [Process] v2.1 - Description
Example: [Marketing] Lead Scoring v2.1 - With AI enrichment
```
